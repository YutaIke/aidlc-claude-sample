## 判断理由

### 実行するステージ

| ステージ | 理由 |
|---------|------|
| アプリケーション設計 | 3つのペルソナ、18のユーザーストーリー、9画面を持つ新規システム。コンポーネント、メソッド、サービスの設計が必要 |
| ユニット生成 | フロントエンド/バックエンド分離により、並行開発可能なユニットへの分解が有益 |
| 機能設計 | 評価計算ロジック（最高・最低点除外平均）など、詳細なビジネスロジック設計が必要 |
| コード生成 | 必須ステージ |
| ビルド＆テスト | 必須ステージ |

### スキップするステージ

| ステージ | 理由 |
|---------|------|
| リバースエンジニアリング | 既存コードなし（グリーンフィールド） |
| NFR要件 | デモ目的で基本的なNFRのみ。要件定義で十分にカバー済み |
| NFR設計 | 複雑なNFRパターン不要 |
| インフラ設計 | ローカル環境デプロイ。クラウドマッピング不要 |
