# Ideation Portal 要件定義書

## 意図分析サマリー

| 項目 | 内容 |
|------|------|
| **プロジェクト名** | Ideation Portal（アイデア提案ポータル） |
| **リクエストタイプ** | 新規プロジェクト |
| **スコープ** | システム全体（複数コンポーネント） |
| **複雑度** | 中程度（デモ目的のシンプルな実装） |
| **目的** | 従業員がイノベーティブなアイデアを提出し、透明な評価・スコアリング・表彰を行うプラットフォーム |

---

## 1. 機能要件

### 1.1 ユーザー管理

| 要件ID | 要件 | 詳細 |
|--------|------|------|
| FR-001 | ユーザー認証 | ユーザー名とパスワードによる認証 |
| FR-002 | ユーザーロール | 3種類のロール：一般従業員、評価パネルメンバー、管理者 |
| FR-003 | ユーザー規模 | 小規模（〜100人）を想定 |

### 1.2 アイデア提出機能

| 要件ID | 要件 | 詳細 |
|--------|------|------|
| FR-004 | 入力フィールド | タイトル、説明文、期待される効果 |
| FR-005 | ドラフト保存 | アイデアを下書きとして保存可能 |
| FR-006 | 画像添付 | 画像ファイルの添付が可能 |
| FR-007 | 公開範囲 | 提出されたアイデアは全員に公開 |

### 1.3 評価フレームワーク

| 要件ID | 要件 | 詳細 |
|--------|------|------|
| FR-008 | スコアリング尺度 | 1〜10点スケール |
| FR-009 | 評価基準 | 実現可能性、インパクト、革新性（均等重み付け） |
| FR-010 | 独立評価 | パネルメンバーは他者のスコアを見ることができない |
| FR-011 | 最終スコア計算 | 最高点と最低点を除外した平均 |
| FR-012 | フィードバック | パネルメンバーはコメント/フィードバックを提供可能 |

### 1.4 アクセス制御

| 要件ID | 要件 | 詳細 |
|--------|------|------|
| FR-013 | パネル任命 | 管理者が手動でパネルメンバーを任命 |
| FR-014 | スコア閲覧 | 提出者は評価コメント/フィードバックも含めて閲覧可能 |

### 1.5 ダッシュボード・リーダーボード

| 要件ID | 要件 | 詳細 |
|--------|------|------|
| FR-015 | 更新方式 | 手動更新 |
| FR-016 | ランキング表示 | 全アイデアの総合ランキング |
| FR-017 | エクスポート | 不要 |

### 1.6 評価サイクル・表彰

| 要件ID | 要件 | 詳細 |
|--------|------|------|
| FR-018 | 評価サイクル | 常時受付・随時評価 |
| FR-019 | 表彰方法 | システム上での表示・通知のみ |
| FR-020 | 表彰対象 | 上位3アイデア |
| FR-021 | 表彰後対応 | 表彰のみで終了（追加アクションなし） |

---

## 2. 非機能要件

### 2.1 技術要件

| 要件ID | 要件 | 詳細 |
|--------|------|------|
| NFR-001 | プラットフォーム | Webアプリケーションのみ |
| NFR-002 | フロントエンド | React.js |
| NFR-003 | バックエンド | Node.js |
| NFR-004 | データベース | SQLite（デモンストレーション目的） |
| NFR-005 | デプロイメント | ローカル環境（デモンストレーション目的） |
| NFR-006 | 外部統合 | 不要（スタンドアロン） |

### 2.2 品質要件

| 要件ID | 要件 | 詳細 |
|--------|------|------|
| NFR-007 | 可用性 | 営業時間内のみ利用可能で問題ない |
| NFR-008 | セキュリティ | 基本的なセキュリティ（HTTPS、パスワード暗号化） |
| NFR-009 | 言語対応 | 日本語のみ |

---

## 3. ユーザーロール定義

### 3.1 一般従業員（アイデア提出者）
- アイデアの提出・編集・削除
- 自分のアイデアの評価結果・フィードバックの閲覧
- リーダーボードの閲覧
- 他の公開アイデアの閲覧

### 3.2 評価パネルメンバー
- 一般従業員のすべての機能
- アイデアの評価（スコアリング）
- フィードバックコメントの入力
- 担当アイデアの評価状況確認

### 3.3 管理者
- 一般従業員・パネルメンバーのすべての機能
- ユーザーの管理（作成・編集・削除）
- パネルメンバーの任命・解除
- ダッシュボードの手動更新
- 表彰の実行

## 4. 画面一覧（想定）

| 画面名 | 概要 | 主なユーザー |
|--------|------|--------------|
| ログイン画面 | 認証画面 | 全ユーザー |
| ダッシュボード | トップ画面、リーダーボード表示 | 全ユーザー |
| アイデア一覧 | 公開アイデアの一覧 | 全ユーザー |
| アイデア詳細 | アイデアの詳細表示 | 全ユーザー |
| アイデア提出 | 新規アイデア入力フォーム | 一般従業員 |
| マイアイデア | 自分の提出アイデア管理 | 一般従業員 |
| 評価画面 | アイデア評価入力 | パネルメンバー |
| ユーザー管理 | ユーザーCRUD | 管理者 |
| パネル管理 | パネルメンバー任命 | 管理者 |

## 5. データエンティティ（想定）

| エンティティ | 主な属性 |
|--------------|----------|
| User | id, username, password, role, created_at |
| Idea | id, title, description, expected_effect, author_id, status, created_at |
| IdeaImage | id, idea_id, file_path, created_at |
| Evaluation | id, idea_id, evaluator_id, feasibility_score, impact_score, innovation_score, comment, created_at |
| Recognition | id, idea_id, rank, recognized_at |

---

## 6. 制約事項・前提条件

1. **デモンストレーション目的**: 本システムはデモ目的で構築され、本番環境での運用は想定していない
2. **SQLite使用**: 軽量なSQLiteを使用するため、同時アクセス数に制限あり
3. **ローカル環境**: 初期はローカル環境での動作を想定
4. **シンプルな評価**: 評価基準は3つ固定（実現可能性、インパクト、革新性）
5. **手動更新**: ダッシュボード・リーダーボードは手動更新

---

## 7. 将来の拡張可能性（スコープ外）

以下の機能は現在のスコープ外だが、将来的に検討可能：

- SSO/OAuth認証対応
- モバイル対応（レスポンシブデザイン）
- データエクスポート機能
- リアルタイム更新
- 多言語対応
- カテゴリ別・部門別ランキング
- 表彰後のワークフロー（実装検討チームへの引き継ぎ等）
